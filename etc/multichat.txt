Sistema de multichat
----------------------

Sistema que permita conversaciones entre más de 2 peers. El sistema de chat directo se
basa en mandar mensajes de texto al peer con el que nso comunicamos, pero este método
no vale para multichat, porque podemos tener varias conversaciones diferentes con un peer.

Debería haber un identificador único de conversación, generado por el que comienza el chat.  Este id
se propaga a todos los que se unan a la conversación. Para mandar un mensaje de chat siempre
se ha de incluir ese id, y el mensaje se transfiere a todos los participantes. Pero puede que
haya participantes que no son amigos entre sí... Se quiere permitir esto???????? Sería útil, sí

Es un poco engorroso establecer conexiones entre peers que no se conocen, es más fácil usar otros peers
como repetidores, pero determinar quién repite a quién no es sencillo. El repetidor podría ser
el que te ha metido en la conversación (que a su vez puede tener otro repetidor). En la mayoría de los
casos esto funcionaría de forma óptima. La conversación enter 2 peers sería un caso particular de
esta.


Este esquema no es válido, porque se rompería en cuanto uno de los que actúan como enlace se desconectase.
Se requiere conexión entre todos los pares. Sedeja para un futuro.
